## 19. 새 세션에서 자주 생기는 실패를 원천 차단

### 19.1 “가격부터 보고 BLIND 작성” 차단

D outcome snapshot URL, header, sha256, row_count, high_return, winner, amount_rank는 blind_seal_receipt 전 접근 금지다. 접근했다면 `blind_valid=false`이고 정상 연구 episode가 아니다.

### 19.2 “Python parse만 하고 연구 안 함” 차단

Python parse는 row/source/outcome population을 만들기 위한 도구다. 이후 material row queue, candidate_screening, 사건 지도, pairwise 비교, red-team이 비어 있으면 연구 미수행이다.

### 19.3 “candidate 20개만 빨리 작성” 차단

final_watchlist가 candidate_screening보다 먼저 생성되면 무효다. candidate_screening은 accepted와 rejected를 모두 포함해야 한다.

다음은 final_watchlist가 뒤에 생성된 것처럼 보여도 무효다.

```text
curated = [...] 에 rank/code/company/source_id를 미리 넣고 candidate_screening을 생성
selected_codes, top_codes, final_codes_order, manual_ticker_list를 먼저 만든 뒤 evidence를 붙임
candidate_screening.proposed_final_rank를 저장한 뒤 그 순서로 final_watchlist 작성
candidate_ranking_audit.jsonl을 final_watchlist 20개에 대해서만 작성하고 나머지 INCLUDE/WATCH 후보의 탈락 사유를 생략
P snapshot high_return/amount leaders를 먼저 뽑고 관련 뉴스만 찾아 final 후보화
keyword 검색 결과만 material universe로 축소
material_reviewed = material_review_queue_member 같은 자동 boolean으로 전수 검토를 가장
```

정상 패턴:

```text
읽다가 떠오른 생각은 provisional_hypothesis.jsonl에 기록
모든 material row review가 닫힌 뒤 candidate_screening 생성

```

### 19.4 “outcome_to_news에 scorecard 섞기” 차단

final scorecard는 별도이다. `outcome_to_news_audit`는 winner census와 1:1이다.

### 19.5 “brain_delta lesson memo” 차단

`lesson`, `takeaway`, `summary` 중심 record만 있으면 brain_delta가 아니다. `record_type`, `source_phase`, `available_from`, `source_fact_ids`, `outcome_audit_ids`, `payload`가 있어야 한다.

### 19.6 “짧은 보고서로 연구한 척” 차단

산문은 압축 가능하다. 하지만 section별 population reference와 JSONL record는 비울 수 없다.

def find_entities(text):
    found=[]; spans=[]
    for a,c in alias_items:
        if not a or len(a)<2: continue
        # avoid single word generic except aliases explicit
        for m in re.finditer(re.escape(a), text):
            # no overlap
            if any(not (m.end()<=s or m.start()>=e) for s,e,_ in spans):
                continue
            rr=resolve_alias(a)
            if not rr: continue
            spans.append((m.start(),m.end(),a))
            found.append({'alias':a,'canonical_name':rr['name'],'ticker':rr['code'],'market':rr['market']})
            break
    # dedupe by ticker
    out=[]; seen_codes=set()
    for x in found:
        if x['ticker'] not in seen_codes:
            seen_codes.add(x['ticker']); out.append(x)
    return out

def has_any(text, terms):
    return any(t in text for t in terms)

def first_sentence_with(text, terms_or_entity=None):
    # split sentences lightly on Korean sentence endings / multiple spaces
    sents = re.split(r'(?<=[다요음임죠\)])\s+|[\n\r]+', text)
    if terms_or_entity:
        for s in sents:
            if any(t in s for t in terms_or_entity):
                return norm_text(s)[:420]
    return norm_text(sents[0] if sents else text)[:420]

def local_predicate(text):
    hits=[p for p in direct_predicates if p in text]

def disposition_for(row, entities, duplicate_of):
    text = row['title']+' '+row['body']
    title=row['title']
    if duplicate_of:

    if has_any(title, notice_terms) or title.startswith('[공시]') or '주요공시' in title:
        return 'DISCLOSURE_OR_MARKET_NOTICE'
    if entities:
        pred=local_predicate(text)
        # title-leading exact issuer or entity + predicate is direct material
        lead_ent = False
        for e in entities:
            if title.startswith(e['canonical_name']) or title.startswith(e['alias']) or f"[{e['canonical_name']}]" in title or f"{e['canonical_name']}," in title[:30] or f"{e['canonical_name']} " in title[:30]:
                lead_ent = True
        if pred and lead_ent:
            return 'DIRECT_ISSUER_MATERIAL'
        if pred and len(entities)<=3:
            return 'DIRECT_ISSUER_SECONDARY'
        # many entities in body might be related links/table audit
        if len(entities)>=5 or '관련기사' in row['body'] or '오늘의 주요뉴스' in title:
            return 'BODY_TABLE_OR_LIST_AUDIT'
        return 'DIRECT_ISSUER_SECONDARY'
    if has_any(text, market_terms):
        return 'MARKET_STATE_REGIME'
    if has_any(text, theme_terms):
        return 'THEME_POLICY_INDUSTRY_EVENT'
    if '상한가' in text or '급등' in text or '거래량' in text or '특징주' in title:
        return 'D1_CONTINUATION_SIGNAL'
    if has_any(text, non_market_terms):
        return 'NON_MARKET_NEWS'
    if len(row['body'])<80:
        return 'LOW_SIGNAL_CONTEXT'
    return 'LOW_SIGNAL_CONTEXT'
